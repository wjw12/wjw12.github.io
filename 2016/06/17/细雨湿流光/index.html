细雨湿流光

发表于 2016-06-17

我编的第一批软件极有诗意，李后主有词云：

红豆啄残鹦鹉粒。

我的软件就曲折和弹性而言，达到了此句的境界。后主又有残句云：

细雨流湿光。

我的软件就有这么简约，别人编十行，我只用一行。等到交活时，教授看了吃一惊：这么短！能跑(run)吗？我说你试试嘛。试完了他和我握手道：谢谢！但是到了开支时，我的钱比别人都少。原来是按行算钱，真把我气死了。

———— 王小波 《革命时期的爱情》

编程也是有美学追求的。这是王小波给人很大启发的一点。简约、优雅的代码不论对编程者本人还是其他阅读修改的人都能带来很大方便。今天我就想介绍一个，能让GPU计算编程十行代码变一行的技术：C++ AMP

C++ AMP是微软为并行异构计算为C++作出的扩展，目前只能在Windows上Visual Studio 2012以上版本使用。对GPU底层操作进行了封装，在不同厂商的显卡上都能用同样的代码运行（显卡需要支持DX11才行）

用一个简单的矩阵乘法作例子。在我的笔记本上GPU比CPU快了大约50倍，而代码相当简短，不用配置一堆环境再加上编写OpenCL的Kernel文件了。一切都完美融入C++语言本身。

#include <amp.h>
using namespace concurrency;
using std::vector;

int main(int argc, char *argv[])
{
    const int M = 1024, N = 1024, W = 1024;
    vector<int> vA(M*W), vB(W*N), vC(M*N);

    /* 初始化元素... */

    array_view<const int, 2> a(M, W, vA), b(W, N, vB);
    array_view<int, 2> c(M, N, vC);
    c.discard_data();

    parallel_for_each(c.extent, [a,b,c](index<2> idx) restrict(amp)
    {
        int row = idx[0]; int col = idx[1];
        int sum = 0;
        for (int i = 0; i < b.extent[0]; i++)
            sum += a(row, i) * b(i, col);
        c[idx] = sum;
    });
    c.synchronize();
}

主要的流程是这样的：首先，存在vector里的数据需要用AMP特定的数组容器array_view储存，相当于把数据存入GPU的global memory；然后，用parallel_for_each函数传递一个lambda函数给每个GPU线程运算，并行程序的主体就放在lambda表达式里面；最后执行synchronize，把数据传回内存，就可以用索引访问array_view对象里的数据了。

parallel_for_each用到了一个特殊的关键词restrict(amp)，是C++ AMP新引入的。有了这个提示，lambda表达式里的语句就会用AMP限定的accelerator(一般是显卡，也可以是别的，可以自行设定)运算，涉及的操作也被限制在GPU能执行的范围内，编译器会做一些检查。

GPU要访问数组里的数据，需要用index这个对象。写起来也和数组索引指标没什么区别，1~3维都可以。

至于GPU计算涉及的种种dirty work，像显存的分配与释放，线程的管理等等，都由AMP自动做了，对我这种拿来主义者实在是解放生产力的好事。至于效率肯定会损失一部分，不过起码比CPU提高了一个数量级不是吗？

更详细的介绍可以见这里https://msdn.microsoft.com/en-us/magazine/hh882446.aspx 本文的代码也来源于此。
