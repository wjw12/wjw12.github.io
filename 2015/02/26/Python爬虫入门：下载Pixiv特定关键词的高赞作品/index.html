Python爬虫入门：下载Pixiv特定关键词的高赞作品

发表于 2015-02-26

刚刚学一点python爬虫，很有趣，可以飙车了（误

原本写了个按关键词下载草榴上的图片的……那个不和谐就不在这里分享了，接下来想下载插画给手机换一换壁纸，P站又不提供按热门度排序搜索图片的功能，于是就想自己实现一个简单的爬虫，下载特定关键词赞数过百的作品（如果手动去找的话要翻几页才能找到一个）

源代码：https://github.com/wjw12/python/blob/master/pixivSpider.py

首先要登陆P站，我这里偷个懒，在IE登陆然后用Fiddler这个软件抓包，用chrome的F12应该也行，抓到cookie内容插入到http request的header里面去就行，免去了cookiejar之类模块的使用

req = urllib.request.Request(pageURL)
req.add_header('Cookie',cookie)
resp = urllib.request.urlopen(req)

编写整个程序的重点在于分析P站的页面逻辑，利用审查元素找到图片、文本在html代码中的位置，找到上下文的规律就可以用正则表达式抓取了，比如说获取赞的数量：

下载图片时一开始总是出现403 error，用抓包软件分析手动登陆时的request，发现爬虫的request少了一个referer，是网页用来检验访客从哪里来的信息。前面在图片页中找到了jpg图片的URL ，直接在浏览器中访问或用python urllib.request.urlopen访问都会出403错误，服务器拒绝访问。只有给我们的request加上一个Referer: http://www.pixiv.net/member_illust.php?mode=medium&illust_id=48620880 伪装成从P站链接进去访问图片才能成功，代码中是这样：

req = urllib.request.Request(imgURL)
referer = imgPageURL
req.add_header('Referer',referer)

要打开图片imgURL必须伪装成从imgPageURL进入的那样，补充阅读：HTTP referer

作为初学者遇到最大的一个麻烦就是字符串编码问题，Python3比起2稍微好了一点，但一开始程序总是各种报错，总结了一些经验：

Python3 里面不加说明打引号的字符串就是unicode，str()返回值都是unicode

Python3 没有unicode()函数，对str进行decode会出错（str类型没有decode方法），对bytes进行encode会出错

unicode和bytes不要在同一语句中混用，比如下面正则表达式替换时，imgPage是从网站读取的页面html，是utf-8编码的bytes字符串，所以pattern和repl都必须encode成为bytes字符串才能用正则匹配，不然会出错

pattern = r'<div>|</div>'.encode('utf-8')
replaceDIV = re.compile(pattern)
repl = '\n'.encode('utf-8')
imgPage = re.sub(replaceDIV,repl,imgPage)

日文汉字在控制台可能无法正确输出，用gb18030 encode之后变成b’\xa3\xb2\xc2’这类形式，到最后还是没法在控制台正确显示文件名，不过下载下来之后在windows里面看到的文件名是正确的
